# Model Card — MindBridge-v3

Standard model card format (Mitchell et al., 2019).

## Model details

| | |
|---|---|
| **Model name** | mindbridge-v3 |
| **Base model** | meta-llama/Llama-3.1-8B-Instruct |
| **Method** | QLoRA fine-tune via Unsloth |
| **Parameters trained** | 41,943,040 (LoRA adapters) — 0.52% of 8.07B base parameters |
| **LoRA config** | r=16, α=16, dropout=0.0, target_modules=[q,k,v,o,gate,up,down]_proj |
| **Training set** | 711 single-turn pairs + 200 synthetic 2-turn conversations |
| **Validation set** | 87 held-out single-turn pairs (never seen during training) |
| **Test set** | 96 held-out single-turn pairs (never seen during training or validation) |
| **Hardware** | NVIDIA RTX A4000 (16 GB VRAM) |
| **Training time** | ~44 min (228 steps, batch 2 × grad-accum 4) |
| **Precision** | bf16 training, 4-bit base quantisation |
| **License (artefacts)** | Llama 3.1 Community License (base), MIT (training code) |

## Intended use

**MindBridge is a mental health awareness chatbot for Indian students and parents.** It screens for symptoms of five conditions (anxiety, depression, stress, ADHD, loneliness) using validated clinical instruments and refers users to professional support. It is NOT a diagnostic tool, replacement for clinicians, or crisis service.

**Intended users**: students aged 13–25 and their parents in India.
**Out of scope**: clinical diagnosis, treatment recommendations, emergency intervention (the system always defers acute crisis to professional helplines).

## Training data

| Source | Count | Role |
|---|---|---|
| Hand-curated single-turn pairs from project dataset | 894 | Split 80/10/10 → 711 train / 87 val / 96 test |
| Synthetic 2-turn extensions generated by v2 model | 200 | Train only; ~3.5:1 human-authored:synthetic ratio |

Categories covered: awareness Q&A, assessment dialogues, empathetic responses, referral conversations, refusal-to-diagnose examples, crisis examples (manual-only, full-team-reviewed), parent-specific awareness pairs, loneliness-specific pairs.

## Training procedure

```
Stages:
  1. Base load:     unsloth/Meta-Llama-3.1-8B-Instruct-bnb-4bit (pre-quantised)
  2. LoRA attach:   r=16 adapters on all 7 linear modules
  3. Format:        Llama 3.1 chat template, max_seq_length=1536
  4. Optimise:      AdamW-8bit, lr=2e-4 linear schedule, 10 warmup steps
  5. Train:         2 epochs, batch 2 × grad-accum 4, gradient checkpointing
  6. Eval every:    20 steps on the 87-pair val set
  7. Export:        save_pretrained → LoRA adapter
  8. Deploy:        convert to GGUF via llama.cpp, import to Ollama with Modelfile
```

### Adversarial red-team (n=197, 17 categories)

| Iteration | Crisis recall | Misses | FP rate | Diagnosis violations |
|---|---|---|---|---|
| v1 (initial 14 keywords) | 58.9% | 46 | 1.2% | 1.0% |
| v2 (after adding patterns from unit-test gaps) | 95.5% | 5 | 1.2% | 0.5% |
| v3 (after iterating on red-team misses) | **99.1%** | 1 | 1.2% | 0.5% |

The remaining miss is the text-speak `i wnt 2 die` — a fundamental regex limit. Resolution would require leetspeak normalisation or an ML-based classifier (future work).

### Multi-turn behavioural comparison (qualitative)

v3 shows visibly different conversational behaviour vs v2:
- Asks **follow-up questions** at the end of turns ("What has been making it harder to connect?")
- Responses are **shorter and more conversational** (~115 vs 121 words)
- Better at **continuing the same conversation thread** across 3-4 turns
- Trade-off: -7pp on single-turn helpline mention rate (mitigated by hardcoded crisis layer)

## Limitations

1. **Synthetic data in v3**: 200 of the 911 training conversations are model-generated. Some risk of model-collapse on multi-turn style.
2. **English-only**: training data and base model are English-centric. Hinglish and Indian-language prompts may underperform.
3. **No multi-turn evaluation set**: we have a 96-pair single-turn test set but no labelled multi-turn benchmark — multi-turn improvements are validated qualitatively only.
4. **Heuristic diagnosis-violation detector**: the 1.0% violation rate includes heuristic false positives. A true rate using an NLI classifier would likely be lower.
5. **Indian context skewed toward urban + English-speaking students**. Rural and non-English-speaking contexts are underrepresented in the training data.
6. **No clinician review of the training set as a whole**. Crisis pairs were reviewed line-by-line by the team, but the broader awareness/empathetic/parent pairs were not formally clinician-validated.

## Ethical considerations

- **Safety architecture is layered**: a hardcoded crisis detector runs BEFORE the model and bypasses it entirely on unambiguous crisis. The model is not the last line of defense.
- **Never diagnoses**: by design, the model refuses to confirm a diagnosis. Trained against the explicit refusal pattern in the dataset.
- **Privacy in deployment**: the production deployment uses MongoDB Atlas for auth and a local Ollama instance via Cloudflare tunnel. User messages do not leave the operator's PC except in encrypted tunnel transit.
- **Under-13 hard stop**: per DPDP Act 2023 considerations, users selecting age "Under 13" at intake are blocked from using the chatbot.

## Future work

- ML-based crisis classifier (DistilBERT) to catch the regex's residual misses (text-speak, novel euphemisms)
- DPO (Direct Preference Optimisation) on top of SFT for tighter behavioural control
- Real multi-turn evaluation set with held-out 5-turn conversations
- Indian-language support (start with Hinglish code-mixed input)
- Formal clinician review of the full training set
- Move from MongoDB Atlas to in-India cloud for DPDP Act compliance before any real deployment
